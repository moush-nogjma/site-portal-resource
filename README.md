# site-portal-resource

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面通常作为资源入口、导航页面或信息展示页使用，不针对任何单个域名或具体网站。仓库的主要目的是提供一个集中存放和版本管理 HTML 静态资源的地方，方便维护与分发。

## 目录结构

```
site-portal-resource/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 公共资源文件（样式表、脚本、图片等）
├── README.md       # 本文件
└── ...
```

- `pages/` 目录下每个子文件夹或文件对应一个独立的页面。
- `assets/` 目录用于存放页面之间共享的静态资源，以减少重复。

## 页面归档说明

每个 HTML 页面均为独立完整的静态页面，不依赖外部服务器或数据库。页面之间无耦合关系，可单独部署或访问。归档时请遵循以下原则：

- 页面文件应放置在 `pages/` 下对应的子目录中（如需）。
- 页面内引用资源请使用相对路径。
- 如有必要，可在页面所在目录中附带 `README.md` 说明该页面的用途或使用方式。

## 维护说明

- 新增页面：在 `pages/` 下创建新目录，放入 HTML 文件及相关资源。
- 更新页面：直接修改对应文件，提交 Pull Request 或直接推送（视仓库权限而定）。
- 删除页面：移除对应目录和文件，并确保无其他页面引用。
- 公共资源修改：注意检查是否影响其他页面。

## 贡献

欢迎提交 Issue 或 Pull Request 来改进页面内容、修复错误或新增页面。请确保提交的 HTML 文件格式规范、无外部恶意链接。

## 许可

本仓库中的内容可自由使用，不设特定许可证限制。如需引用，请标注来源仓库。
